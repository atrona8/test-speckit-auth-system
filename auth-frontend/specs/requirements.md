# Auth Frontend

## Objectif
Créer une interface utilisateur simple pour le système d'authentification.

## Fonctionnalités

REQ-001 : L'utilisateur voit un formulaire d'inscription (email + mot de passe)  
REQ-002 : L'utilisateur voit un formulaire de connexion (email + mot de passe)  
REQ-003 : L'utilisateur peut se déconnecter depuis le dashboard  
REQ-004 : Les erreurs API sont affichées à l'utilisateur  
REQ-005 : Le token JWT est stocké et utilisé pour les requêtes authentifiées  
REQ-006 : L'utilisateur est redirigé automatiquement selon son état de connexion  

## Contraintes

- Interface simple, sans framework JS
- Communication avec l'API backend sur http://localhost:3000
- Vanilla HTML + CSS + JavaScript uniquement
